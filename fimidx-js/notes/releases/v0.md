# `fimidx` or `fimidx-js` `v0.x.x` - Release Notes

## v0.5.0

- Made some changes to how log functions from `FimidxConsoleLikeLogger` generates the log object passed to `fimidx`.

  ```typescript
  fimidxConsoleLogger.info();
  {message: "Empty message"};

  // strings or any other primitives are stringified using JSON.stringify(...)
  fimidxConsoleLogger.info("Message");
  {message: "Message"};

  fimidxConsoleLogger.info("Message", {obj: true});
  {message: "Message", obj: true};

  fimidxConsoleLogger.info("Message", {obj: true}, {secondary: true}, {tertiary: true});
  {message: "Message", remainingArgs: [{obj: true}, {secondary: true}, {tertiary: true}]};
  ```

- Added `debug` constructor param to `FimidxNextAuthLogger` besides the required `FimidxConsoleLikeLogger` argument, to inform the logger whether to log debug messages or not. This is inline with how `next-auth` logs messages.
